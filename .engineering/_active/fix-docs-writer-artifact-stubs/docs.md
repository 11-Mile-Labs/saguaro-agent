# Docs-writer Artifact Quality Guard

## Summary

This change closes a workflow quality gap where a docs-writer phase could be
marked complete while the persisted phase artifact contained only a pointer to
another file or a very short placeholder. Saguaro workflow closeout depends on
artifacts being durable and inspectable inside the run directory, so docs-writer
output now has two layers of protection: stronger dispatch instructions at
envelope generation time and server-side validation when complete docs-writer
artifacts are recorded.

The dispatch contract generated for a docs-writer phase now tells the host model
to produce complete multi-section markdown prose and to pass that full markdown
document through `artifact.content` when calling `workflow_record_artifact`.
This keeps the required behavior visible at the moment a harness or agent is
executing the phase, without changing workflow YAML schema or requiring bundled
workflows to carry duplicate prose in every docs phase definition.

The workflow server now validates complete docs-writer artifacts before writing
files or advancing run state. The guard applies only when the phase agent is
`docs-writer` and the artifact status is `complete`. It rejects known
pointer-only wording and rejects artifacts with fewer than 1024 non-whitespace
characters. Partial and failed docs artifacts are left alone so troubleshooting
output can still be recorded, and non-docs phases keep their existing behavior.

## Implementation Notes

`mcp-servers/core/src/workflow/envelope.ts` gained a docs-writer phase check and
two additional dispatch contract sentences. The new text is intentionally
narrow: it explains that docs-writer artifacts must be complete multi-section
markdown, not a pointer, summary, or path, and that
`workflow_record_artifact` must receive the full markdown in `artifact.content`.

`mcp-servers/saguaro-workflow/src/server.ts` gained named constants for the
minimum complete docs-writer artifact size and pointer-only marker patterns.
`workflowRecordArtifact` now resolves the workflow phase from persisted run
state, normalizes the payload, and calls the guard before computing the artifact
path, writing content, or recording phase completion. That means a bad complete
docs artifact fails without mutating the run.

`mcp-servers/saguaro-workflow/src/__tests__/tools.test.ts` gained a small
docs-focused workflow fixture. Regression tests cover the dispatch text,
pointer-only artifact rejection, undersized artifact rejection, and a positive
case proving complete markdown prose still records successfully. The positive
fixture is longer than the new threshold and contains multiple sections so it
behaves like a realistic documentation artifact rather than a token placeholder.

## Verification

The targeted workflow package test suite passed with 3 test files and 12 tests.
The workflow package typecheck also passed. `git diff --check` reported no
whitespace errors. A privacy scan over touched source, tests, and ticket
artifacts found no private downstream project names in source or test fixtures;
matches on TypeScript access modifiers are not project references.

One operational note surfaced during the dogfood run: the currently running
Saguaro MCP server was loaded before this source patch, so its live
`workflow_dispatch_phase` response for this run still showed the old docs-writer
dispatch text. The source-level regression test proves the new contract is
generated by patched code. Active MCP tooling will reflect the updated dispatch
text after the server is rebuilt or reloaded.

## Knowledge Captures

- Docs-writer dispatch envelopes now require complete multi-section markdown
  prose and full markdown `artifact.content`.
- Complete docs-writer artifact recording now rejects pointer-only content and
  undersized markdown before persistence or run advancement.
- The guard is scoped to complete docs-writer phases, preserving existing
  behavior for other phase types and non-complete docs artifacts.
- Running MCP servers may need reload or rebuild before live tool responses
  reflect source changes made during an active dogfood run.
